‚öôÔ∏è How It Works

Game Loading Process:

  1. Library page loads ‚Üí CloudGameManager.getAllGames()
  2. Fetch local games from gamefolder/
  3. Try to fetch cloud games from Supabase API
  4. If cloud fails ‚Üí graceful fallback to local only
  5. Combine and display all games with source indicators

Import Process (When Supabase is set up):

  1. User creates game in Builder
  2. Clicks "Upload to Cloud" button
  3. Game data sent to /api/games endpoint
  4. Stored in Supabase database
  5. Assets uploaded to Supabase Storage
  6. Immediately available to all users globally

üöÄ Setup Guide

To enable real cloud functionality:

  1. Create Supabase account and project
  2. Run database/schema.sql in Supabase SQL Editor
  3. Set environment variables in Vercel:
    • NEXT_PUBLIC_SUPABASE_URL
    • NEXT_PUBLIC_SUPABASE_ANON_KEY
  4. Deploy to Vercel (API endpoints won't work locally)
  5. Games uploaded by users will be available globally!
